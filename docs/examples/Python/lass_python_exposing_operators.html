Exposing operators

This tutorial shows how you can make your exported class support operators and special functions like len. The philosophy employed is that the special member functions like the operators should use a similar way of exporting as the regular functions. The name of the function to be given in Python is used to do that. Instead of using a C-string you can use one of the special identifiers available in lass::python to map your function onto these special functions and operators. The list of available identifiers is available in the namespace lass::python::methods. All of these identifiers are surrounded with underscores, not double underscores as that is typically reserved for internal use of the C/C++ compiler folks. The list of identifiers is:

Special methods:

_call_, _repr_, _str_

Methods and operators supporting the number protocol:

Unary operators: _neg_, _pos_, _abs_, _invert_, _int_, _long_, _float_
Unary operators only supported in Python version 2.x: _hex_, _oct_

Binary operators: _add_, _sub_, _mul_, _mod_, _lshift_, _rshift_, _and_, _or_, _xor_

Binary in-place operators: _iadd_, _isub_, _imul_, _imod_, _ilshift_, _irshift_, _iand_, _ior_, _ixor_

Binary division operators: _truediv_, _itruediv_, _floordiv_, _ifloordiv_

Binary division operators only supported in Python version 2.x: _div_, _idiv_

Ternary operators: _pow_

Comparators

_lt_, _le_, _eq_, _ne_, _gt_, _ge_

Methods and operators for mapping and sequence protocols:

Methods shared between mapping and sequence protocols: _len_, _getitem_, _setitem_. You can use these identifiers if you don't care on which protocol your function is mapped to. The sequence protocol will get priority when you use any of these identifiers.

Methods for the sequence protocol: seq_len_, _concat_, _repeat_, seq_getitem, _getslice_, seq_setitem_, _setslice_, _contains_, _iconcat_, irepeat_

Methods for the mapping protocol: map_len, map_getitem_, map_setitem_

Methods and operator for supporting the iterator protocol

_iter_, next
